# Accessibility review

1. Compare relative luminance for colours used on adjacent thin marks.
2. Inspect a grayscale rendering and add line style or marker redundancy where ordering collapses.
3. Simulate common colour-vision deficiencies with a trusted tool when available.
4. Check marks at final publication size, including legends and uncertainty bands.
5. Verify text and annotations against their actual backgrounds.
6. Avoid encoding the only distinction with red versus green.

Report limitations explicitly. Accessibility is a property of the complete encoding, not a HEX list.
